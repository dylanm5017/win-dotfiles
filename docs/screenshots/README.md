# Screenshot checklist

The main `README.md` has a **Screenshots** section with commented-out `![...]` tags pointing at the
files below. Capture each with `Win+Shift+S`, save as PNG into this folder using the exact filenames,
then uncomment the matching tags in the root `README.md`.

- [ ] `desktop.png` — full desktop: both Yasb bars visible, komorebi tiling a few windows.
- [ ] `terminal.png` — Windows Terminal or WezTerm on the active theme's color scheme.
- [ ] `flow-launcher.png` — Flow Launcher open with a few themed results showing.
- [ ] `context-menu.png` — the Nilesoft Shell–themed right-click context menu.
- [ ] `theme-comparison.png` *(optional)* — the same view (e.g. the desktop) across 2-3 themes, to show
      off the live `theme <name>` switching.

Tips:
- Pick a theme with good visual contrast for the first pass (e.g. `gruvbox-material` or `mocha`) —
  switch with `theme <name>` first if needed.
- Crop tightly; a full 4K desktop screenshot scaled down in a README is often too small to read. A
  bar/taskbar strip or a cropped window is usually more legible than a full-screen capture.
- Keep file sizes reasonable (a few hundred KB each) — compress before committing if a capture tool
  saves an uncompressed PNG.
